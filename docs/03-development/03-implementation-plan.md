### 1.2 技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端框架 | React + Vite | 组件化开发，TRAE IDE 支持最佳 |
| 天空渲染 | Canvas 2D / Three.js | 实时天空效果，粒子系统 |
| 地图可视化 | ECharts 地图组件 | 轻量，支持中国地图 |
| 图表 | ECharts | 功能丰富，易用 |
| 后端 | Hono (Edge) | 轻量，可部署到 Cloudflare Workers |
| 数据库 | Turso (libSQL) / SQLite | 免费额度足够，边缘部署 |
| 音效 | Web Audio API | 无需外部依赖，纯前端实现 |
| 部署 | Vercel / Cloudflare Pages | 免费托管，零成本 |

### 3.2 核心模块职责

| 模块 | 功能 | 技术要点 |
|------|------|----------|
| `skyEngine.js` | 天空渲染引擎 | Canvas 2D；8种天气 shader 预设；粒子系统（雨滴/雪花/阳光）；城市天际线 SVG 剪影；平滑过渡动画（lerp 插值） |
| `audioEngine.js` | 环境音效 | Web Audio API 程序化生成（白噪音 + 滤波器模拟雨声；振荡器模拟雷声）；无需加载音频文件 |
| `dataManager.js` | 数据管理 | 预置模拟数据（50+条，覆盖10+城市）；localStorage 读写真实用户数据；合并显示逻辑 |
| `emotionMapper.js` | 情绪-天气映射 | 8种情绪到天气的映射规则；混合天气计算逻辑 |

### 3.3 情绪-天气映射规则

| 情绪 | 天气 | 权重 |
|------|------|------|
| 开心 | 晴天 | 1.0 |
| 平静 | 多云 | 0.8 |
| 焦虑 | 小雨 | 0.7 |
| 悲伤 | 大雨 | 0.9 |
| 愤怒 | 雷暴 | 1.0 |
| 迷茫 | 雾霾 | 0.7 |
| 疲惫 | 黄昏 | 0.6 |
| 充满活力 | 极光 | 0.9 |

### 3.4 区域聚合算法

```
1. 获取区域内最近 50 条情绪数据
2. 按情绪类型分组统计数量
3. 计算每种情绪的权重占比
4. 选择权重最高的情绪作为主导天气
5. 如果次要情绪占比 > 20%，生成混合天气描述
6. 示例：60%开心 + 30%平静 + 10%焦虑 → "晴间多云，偶有微风细雨"
```

## 5. TRAE Code 模式快速启动指令

### 5.1 第一阶段：项目骨架

```
我要开发一个叫"情绪天空"的 Web 应用，参加 TRAE AI 创造力大赛生活娱乐赛道。

## 核心功能
1. 情绪选择器：用户从 8 种情绪中选择当前心情（开心/平静/焦虑/悲伤/愤怒/迷茫/疲惫/充满活力）
2. 天空渲染引擎：根据情绪渲染对应的天气场景（晴天/多云/小雨/大雨/雷暴/雾霾/黄昏/极光），包含：
   - 动态天空背景（Canvas 渐变 + 粒子效果）
   - 赛博朋克城市天际线剪影（前景）
   - 天气粒子动效（雨滴/雪花/阳光粒子/闪电/雾气/极光带）
   - 环境音效（Web Audio API 程序化生成）
3. 城市情绪地图：展示中国各城市的当前"情绪天气"，使用 ECharts 地图组件
4. 情绪时间线：展示个人情绪记录和区域情绪趋势

## 技术方案
- 纯前端方案（React + Vite），数据存 localStorage
- 预置 50+ 条模拟数据覆盖 10+ 城市，同时存储真实用户提交
- 天空渲染用 Canvas 2D（如时间不够则不用 Three.js）
- 城市地图用 ECharts 中国地图
- 音效用 Web Audio API 程序化生成

## 页面结构
- 首页：情绪选择器 + 个人天空场景（全屏沉浸式）
- 地图页：城市情绪天气地图 + 点击展开详情
- 趋势页：个人/区域情绪时间线图表
- 导航：底部 tab 切换

## 设计风格
- 赛博朋克 + 数据可视化美学
- 深色背景（#0a0e1a）
- 强调色：天蓝色(#38bdf8)、紫色(#a78bfa)、粉色(#f472b6)

请先搭建项目骨架，配置好开发环境。
```
